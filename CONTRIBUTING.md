# Contributing to Aegis Vault

First off, thank you for considering contributing to Aegis Vault! It's people like you that make the Stacks ecosystem such a great place.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs

- Check the GitHub Issues to see if the bug has already been reported.
- If not, create a new issue with a clear title and description.
- Include as much relevant information as possible, including steps to reproduce.

### Suggesting Enhancements

- Open a GitHub Issue to discuss your idea.
- Provide a clear explanation of the feature and its benefits.

### Pull Requests

- Fork the repository and create your branch from `main`.
- Ensure your code follows the existing style and pass all tests.
- Each PR should be atomic and focused on a single improvement.
- All commits must be GPG/SSH signed.

## Development Setup

See the [README.md](./README.md#setup) for detailed installation and setup instructions.

### Smart Contracts
- Use `clarinet check` and `clarinet test` frequently.
- Add comments to any complex logic.

### Frontend
- Use `npm run frontend:dev` for local development.
- Ensure all new components are accessible (ARIA labels, roles).

## Questions?

Feel free to open an issue or reach out to the maintainers.
